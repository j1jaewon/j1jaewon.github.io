# TBT 심층분석보고서 자동 초안 생성 시스템 도입 제안

**수신**: 수출지원센터장  
**발신**: 수출지원센터 정재원 연구원  
**일자**: 2026년 6월  
**제목**: WTO TBT 통보문 기반 보고서 초안 자동화 파이프라인 도입 제안

---

## 1. 현황 및 문제

현재 TBT 심층분석보고서 1건 작성에는 **번역(0.5일) + 구조화(0.5일) + 초안(1일) = 약 2일**이 소요됩니다.  
주당 7–8건 처리 시 담당자 1인이 보고서 작성에만 집중해야 하며, 분석·대외 업무 시간이 부족합니다.

---

## 2. 제안 내용

**WTO TBT 공개 통보문 → AI 번역·추출 → KTL 서식 초안 자동 생성** 파이프라인 도입

| 단계 | 현재 (수작업) | 제안 (자동화) |
|------|--------------|---------------|
| 원문 수집 | ePing 수동 검색 | API 자동 수집 |
| 번역 | ChatGPT 수동 붙여넣기 | Claude API 자동 번역 |
| 서식 적용 | 양식 수동 채우기 | master template 자동 매핑 |
| 초안 출력 | 1–2시간 | **5분 이내** |
| 담당자 역할 | 번역 + 구조화 + 초안 | **검토·검증·승인만** |

---

## 3. 기대 효과

- **처리 속도**: 초안 생성 시간 약 90% 단축 (2일 → 0.5일)
- **품질**: AI가 master template 서식을 강제 적용 → 양식 불일치 오류 감소
- **Human-in-the-loop 유지**: AI는 초안만 생성, 최종 검토·승인은 담당자 → 공공기관 신뢰성 요건 충족
- **누락 방지**: 의견수렴 기간 내 TBT 통보문 자동 감지 → 대응 기간 확보

---

## 4. 구현 방안 (단계적)

**Phase 1 (즉시 가능)**: WTO TBT 공개 API → 초안 마크다운 자동 생성  
→ 내부 검토 후 HWP 붙여넣기 방식 운영 (추가 시스템 불필요)

**Phase 2 (IT 협의 후)**: KTL 내부망 연동 → Dooray 메일 C2P 통보문까지 커버  
→ 국내 관보 및 WTO 미통보 규제도 자동 감지

**보안**: 공개 API만 사용, 내부 데이터·PII 없음 (Phase 1 기준)

---

## 5. 파이프라인 구현 현황 (샌드박스)

`scripts/tbt_report_pipeline.py` — WTO ePing API + Claude API 기반 작동 확인  
`output/sandbox-reports/` — 자동 생성 보고서 초안 샘플

---

## 6. 요청 사항

1. **ANTHROPIC_API_KEY** GitHub Secrets 등록 승인 (월 예상 비용: 건당 약 $0.05)
2. 시범 운영 기간(4주) 동안 실제 보고서 1건 적용 후 품질 평가

> *본 제안의 파이프라인은 공개 데이터만 사용하며, 내부 데이터·PII는 처리하지 않습니다.*
